一、组件化开发

在组件化开发中，保持每个 Vue 组件的代码简洁和易于维护是很重要的。
通常建议每个 Vue 组件的代码控制在 200 行以内。
如果组件的代码超过了这个范围，可能需要考虑将其拆分为更小的子组件。这样可以提高代码的可读性和可维护性。
不需要考虑组件的复用，如果一个组件需要数据，那应该由它本身发起请求，在主vue文件中引入这个组件即可
参照：Doctor.vue 与 其组件 components/doctor/DoctorMap.vue 和 DoctorTable.vue的设计

二、请求
请求都采用异步请求 .vue中调用后 得到response json格式类型
response 的结构是：
{
    "data": 数据值，
    "message":"后端传入的一些信息"，
    "code": 数字值
}

data部分，前端和后端要商议好数据的格式，在能完成功能的情况下，尽可能的简化传输数据

code值与信息对应情况：
1. 200 正常
2. 404 未找到
3. 500 服务器错误
4. 403 禁止访问
5. 400 请求错误
6. 401 未授权
7. 301 永久重定向
8. 302 临时重定向
9. 304 未修改
10. 503 服务器不可用
11. 504 网关超时
12. 502 网关错误
13. 400 请求错误
